Privacy

Last updated: 2026-05-25

catm is designed so that everything happens on your device. No accounts, no servers, no telemetry. This page is the formal statement of that — for both the web app at catm-app.github.io and the Send to catm Chrome extension.

Where things run

catm is a single-page web app served from GitHub Pages. After your first visit:

  • The app shell is cached locally as a Progressive Web App.
  • The Kokoro 82M text-to-speech model is downloaded once from huggingface.co and stored in your browser's HTTP cache (~310 MB).
  • Synthesis runs in your browser via ONNX Runtime Web — WebGPU when available, multi-threaded WASM otherwise.
  • Text you paste, saved sessions, and the audio they produce live only on your device: session metadata in IndexedDB, audio fragments in the Origin Private File System (OPFS).

catm has no backend. No request ever leaves your browser carrying your text or your audio.

Network requests catm makes

  • Static app files from catm-app.github.io (GitHub Pages).
  • The one-time download of the Kokoro model from huggingface.co.
  • Nothing else.

There are no analytics, no error-reporting services, no third-party scripts, and no trackers.

The "Send to catm" Chrome extension

The extension exists only to make it easier to send text you are already reading to catm. It does these things:

  • Adds a context-menu entry, Send selection to catm, that appears when you right-click on selected text.
  • On click, opens catm-app.github.io with the selected text passed as a URL query parameter — along with the active tab's title and URL, so catm can show useful context.
  • Clicking the toolbar icon opens (or focuses) catm.

It does not:

  • Read any page contents on its own. The extension has no content scripts and no broad host permission. It only sees text you yourself select and then explicitly send via the context menu.
  • Communicate with any server. The selected text is passed to a new tab as a URL parameter; the extension itself makes no network requests.
  • Store anything. The extension does not use chrome.storage, cookies, or any other persistent state.
  • Sync anything between devices.

The only permissions it requests are contextMenus plus host access to https://catm-app.github.io/* — the latter so it can find and re-use an existing catm tab instead of opening duplicates.

How to delete your data

  • Inside catm: use the Delete everything button. It purges IndexedDB sessions, OPFS audio, your onboarding and voice preferences, and the cached Kokoro model.
  • From the browser: clear site data for catm-app.github.io in your browser settings.
  • The extension: uninstall it from chrome://extensions. It has nothing else to clean up.

Children

catm is general-purpose. It does not knowingly collect personal information from children — because it does not collect personal information from anyone.
